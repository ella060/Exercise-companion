## 1. 产品概述

### 1.1 产品定位

一款面向中老年女性的居家健身陪伴应用，主打**精神陪伴 + 简单健身指导**，通过温暖的话语、轻松的健身方式和社交分享功能，帮助用户在家中建立规律的运动习惯。

### 1.2 目标用户

- **主要用户**：50-70岁的中老年女性
- **用户特征**：
  - 喜欢广场舞、太极、瑜伽、散步等轻度运动
  - 使用微信为主要社交工具
  - 对手机操作有一定基础但不太熟悉复杂功能
  - 渴望被关心、容易受温暖话语鼓励

### 1.3 核心价值

- **陪伴感**：每日关怀消息 + 温暖UI设计，让用户感到被关注
- **成就感**：打卡记录 + 海报分享，激发持续运动动力
- **实用性**：视频收藏 + 就医提醒，一站式满足运动和健康需求

## 3. 详细功能说明

### 3.1 首页（Home）

**功能描述**：用户打开App后的第一个页面，展示核心数据和引导入口。

**页面元素**：

| 元素           | 说明                                           |
|----------------|------------------------------------------------|
| 头像 + 欢迎语  | 显示"早上好/下午好，今天也要加油哦"            |
| 当前日期       | 格式：2026年5月3日 星期日                      |
| 快捷入口卡片   | 视频库、每日打卡两个主要入口                   |
| 统计数据       | 连续打卡天数、累计运动分钟                     |
| 关怀消息       | 根据时间段推送不同的温暖话语                   |

**关怀消息规则**：

| 时间段   | 消息内容示例                                       |
|----------|----------------------------------------------------|
| 08:59前  | 早上好！新的一天开始啦，今天也要加油哦~            |
| 09:00-11:59 | 今天天气不错，适当活动一下对身体好哦~            |
| 12:00-13:59 | 中午好，饭后散散步有助于消化~                   |
| 14:00-17:59 | 记得多喝水，运动后要补充水分哦                   |
| 18:00-19:59 | 晚上好，适当的夜练可以助眠哦~                   |
| 20:00-21:59 | 今天你已经很棒了，休息一下也没关系的             |
| 22:00后   | 夜深了，早点休息，明天继续加油！                  |

### 3.2 视频库（Video）

**功能描述**：收藏和播放来自各平台的健身视频链接。

**支持平台**：
- 抖音
- B站（哔哩哔哩）
- 微信视频号
- 其他可跳转的网页链接

**视频信息**：

| 字段     | 类型   | 说明                 |
|----------|--------|----------------------|
| 链接     | string | 视频的完整URL        |
| 标题     | string | 用户自定义名称       |
| 分类     | enum   | 广场舞/瑜伽/太极/拉伸/散步/其他 |

**交互说明**：
- 点击"播放"按钮 → 在新窗口打开视频链接
- 支持删除已添加的视频

### 3.3 每日打卡（Check-in）

**功能描述**：记录每日运动情况，支持查看历史打卡日历。

**打卡表单**：

| 字段       | 类型     | 必填 | 说明                       |
|------------|----------|------|----------------------------|
| 运动时长   | number   | 是   | 单位：分钟，最小1          |
| 运动类型   | select   | 是   | 散步/广场舞/瑜伽/太极/拉伸/其他 |
| 今日感想   | textarea | 否   | 最多200字                  |

**日历组件**：
- 显示当前月份的日历
- 已打卡日期高亮显示
- 今日标记特殊样式
- 支持切换上/下月份

**统计数据计算**：
- **连续打卡天数**：从今天往前推算，连续打卡的天数（中断则归零）
- **累计运动分钟**：所有打卡记录的运动时长总和

### 3.4 打卡海报（Poster）

**功能描述**：基于当日打卡数据生成精美的分享海报。

**海报内容**：

| 元素         | 说明                     |
|--------------|--------------------------|
| 标题         | 每日运动打卡             |
| 日期         | 格式：2026年5月3日       |
| 连续天数     | 大字体显示              |
| 运动时长     | XX分钟                   |
| 运动类型     | 广场舞/瑜伽等            |
| 底部标语     | 运动陪伴 · 每天进步一点点 |

**海报样式**：提供4种渐变色模板，用户可切换选择。

**分享功能**：
- 点击"分享海报"→ 提示用户长按保存或直接分享

### 3.5 就医待办（Medical）

**功能描述**：记录就医/康复复查的待办事项。

**记录字段**：

| 字段       | 类型     | 必填 | 说明                 |
|------------|----------|------|----------------------|
| 就医日期   | date     | 是   | 默认填明天           |
| 医院名称   | text     | 是   | 如：市人民医院       |
| 科室       | text     | 否   | 如：骨科             |
| 备注       | textarea | 否   | 注意事项，如：空腹   |

**交互说明**：
- 点击记录卡片 → 切换完成/未完成状态
- 支持删除记录
- 列表按日期升序排列

### 3.6 个人中心（Profile）

**功能描述**：查看个人信息、App设置。

**页面元素**：

| 元素           | 说明                 |
|----------------|----------------------|
| 头像           | 默认🏃‍♀️              |
| 昵称           | 固定"运动爱好者"     |
| 累计天数       | 与首页统计数据同步   |
| 关于我们       | 显示App版本信息       |
| 清除数据       | 一键清除所有本地数据 |

## 4. 数据存储

### 4.1 本地存储（localStorage）

| Key               | 数据类型 | 说明                 |
|-------------------|----------|----------------------|
| fitness_videos    | array    | 视频列表             |
| fitness_checkins | array    | 打卡记录列表         |
| fitness_medicals | array    | 就医记录列表         |
| fitness_stats     | object   | 统计数据缓存         |

### 4.2 后端数据库（SQLite）

**表结构**：

```sql
-- 打卡记录
CREATE TABLE checkins (
    id INTEGER PRIMARY KEY,
    date TEXT UNIQUE,      -- 日期，格式：YYYY-MM-DD
    minutes INTEGER,       -- 运动时长（分钟）
    type TEXT,             -- 运动类型
    note TEXT,             -- 感想备注
    created_at TEXT        -- 创建时间
);

-- 视频收藏
CREATE TABLE videos (
    id INTEGER PRIMARY KEY,
    url TEXT,
    title TEXT,
    category TEXT,
    created_at TEXT
);

-- 就医记录
CREATE TABLE medicals (
    id INTEGER PRIMARY KEY,
    date TEXT,
    hospital TEXT,
    department TEXT,
    note TEXT,
    done INTEGER DEFAULT 0,
    created_at TEXT
);

-- 统计数据
CREATE TABLE stats (
    id INTEGER PRIMARY KEY,
    streak INTEGER,         -- 连续打卡天数
    total_minutes INTEGER,  -- 累计运动分钟
    updated_at TEXT
);

-- 关怀消息记录
CREATE TABLE caring_messages (
    id INTEGER PRIMARY KEY,
    content TEXT,
    sent_at TEXT
);
```

## 7. 非功能性需求

### 7.1 性能需求

- 页面首屏加载时间 < 2秒
- 操作响应时间 < 500ms
- 支持离线查看（本地数据缓存）

### 7.2 兼容性需求

- 移动端优先（iOS Safari、Android Chrome）
- 适配手机屏幕（320px - 428px宽度）
- 禁用缩放（maximum-scale=1.0）

### 7.3 数据安全

- 本地数据仅存储在用户设备
- 不收集任何个人隐私信息
- 清除数据为不可逆操作，需二次确认
